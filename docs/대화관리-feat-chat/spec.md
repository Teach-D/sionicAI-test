# 기능 명세서 — 대화(Chat) 관리

- **브랜치**: `feat/chat`
- **작성일**: 2026-07-14

---

## 1. 개요

사용자가 질문을 입력하면 OpenAI Chat Completions API를 통해 답변을 생성하고, 그 대화 쌍을 저장·조회·삭제하는 기능을 제공한다.
대화는 스레드(Thread) 단위로 묶이며, 동일 스레드 내의 이전 대화 전체가 OpenAI 요청의 컨텍스트로 함께 전송된다.

---

## 2. 도메인 모델

엔티티 필드·DB 타입·인덱스는 [erd.md](./erd.md)를 단일 출처로 한다.

### 스레드(Thread) 생성 규칙

아래 조건 중 하나라도 해당하면 새 스레드를 생성한다:

1. 해당 유저의 첫 번째 질문인 경우
2. 유저의 마지막 질문 시각(`threads.last_chat_at`)으로부터 30분이 경과한 경우

위 조건에 해당하지 않으면 가장 최근 스레드를 재사용한다.

### OpenAI 컨텍스트 구성

- 현재 스레드에 속한 **모든 이전 대화**(question/answer 쌍)를 `messages` 배열에 순서대로 포함한다.
- system 프롬프트는 설정 파일(`application.yml`)에서 주입한다.

엔드포인트 상세·요청/응답 스키마·에러 코드·권한 테이블은 [api.md](./api.md)를 단일 출처로 한다.

## 4. 구현 범위

### In-scope

- `Thread` 엔티티 및 `ThreadRepository`
- `Chat` 엔티티 및 `ChatRepository`
- 대화 생성 (`POST /api/chats`) — 일반 응답 및 SSE 스트리밍 응답
- 스레드·대화 목록 조회 (`GET /api/threads`) — 스레드 단위 그룹화, 정렬, 페이지네이션
- 스레드 삭제 (`DELETE /api/threads/{threadId}`) — 하위 Chat cascade 삭제
- 스레드 30분 분기 로직
- OpenAI Chat Completions 연동 (일반·스트리밍)

### Out-of-scope

- 개별 Chat 삭제 (스레드 단위 삭제만 제공)
- 스레드 제목/이름 관리
- 대화 수정
- OpenAI 외 모델 공급자 연동
- 토큰 사용량 추적 및 비용 관리

---

## 5. 스트리밍 처리

- `isStreaming=true` 시 응답 형식은 `text/event-stream` (SSE).
- 스트리밍 중 OpenAI 오류 발생 시:
  - 스트리밍 시작 전 오류 → 일반 에러 응답 반환.
  - 스트리밍 시작 후 오류 → `data: [ERROR]` 이벤트를 전송하고 스트림 종료.
- 스트리밍이 완료된 후 누적된 전체 답변을 DB에 저장한다.
