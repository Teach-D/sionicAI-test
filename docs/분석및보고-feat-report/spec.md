# 기능 명세서 — 분석 및 보고

- **브랜치**: `feat/report`
- **작성일**: 2026-07-14

---

## 1. 개요

관리자가 서비스 운영 현황을 파악할 수 있도록 사용자 활동 집계 및 대화 보고서 생성 기능을 제공한다.
활동 기록 조회는 요청 시점 기준 24시간 내 회원가입·로그인·대화 생성 수를 반환하고, 보고서 생성은 같은 기간의 전체 사용자 대화 목록을 CSV 파일로 다운로드할 수 있다.
두 기능 모두 `ADMIN` 역할을 가진 사용자만 접근 가능하다.

---

## 2. 도메인 모델

엔티티 필드·DB 타입·인덱스는 [erd.md](./erd.md)를 단일 출처로 한다.

### 활동 이벤트 타입

| 이벤트 타입 | 발생 시점 |
|---|---|
| `SIGN_UP` | 회원가입 완료 시 |
| `LOGIN` | 로그인 성공 시 |
| `CHAT_CREATED` | 대화(Chat) 생성 완료 시 |

### 집계 범위

- "하루 동안"은 요청 시점 기준 **직전 24시간** (rolling window).
- 집계 기준: `created_at >= (now − 24h) AND created_at <= now`

## 4. 구현 범위

### In-scope

- `ActivityLog` 엔티티 및 `ActivityLogRepository`
- 회원가입·로그인·대화 생성 시 `ActivityLog` 저장 (기존 서비스 코드에 최소 수정)
- 활동 기록 조회 (`GET /api/admin/activity`) — 24시간 내 이벤트 타입별 카운트 반환
- CSV 보고서 생성 (`GET /api/admin/report/csv`) — 24시간 내 전체 대화 목록을 CSV 파일로 다운로드

### Out-of-scope

- 특정 사용자 개인의 활동 기록 조회 (전체 집계만 제공)
- 24시간 외 다른 기간 범위 지원
- 보고서 파일의 서버 저장 또는 이메일 전송
- 실시간 대시보드 또는 차트 데이터 API
- 대화 내용(질문/답변) 외 다른 데이터의 CSV 포함
